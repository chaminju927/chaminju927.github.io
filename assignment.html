1. Http? Html?

Http(Hyper-text transfer protocol)이란, html의 문서를 원활하게 받고 다른 html문서를 찾아가기 쉽도록 만든 통신 프로토콜이다.

Http 등장 이전에는 파일주고 받거나 통신할때 FTP(File Transfer Protocol)이라는 프로토콜을 이용했는데, 이는 논문이 참조하는 다른 논문을 다운로드
받기가 어렵고 논문 안에 다른 논문이 업로드 된 서버 주소가 없다는 문제가 있었다.

따라서 텍스트에 논문의 위치정보와 그림, 영상 등을 삽입하고 텍스트의 포맷을 지정할수 있는 Hyper-Text가 등장하였고, 이를 설명하고 제어하는 언어인 HTML(Hyper-Text Markup Language)이 등장한 것이다.


2. 시맨틱 태그란?

2-1. 시맨틱 태그의 목적

시맨틱 태그란 의미가 있는 태그를 말한다. header, main, footer, section, article, aside 태그가 이에 해당한다. 이런 태그를 사용하는 목적은 다음과 같다.
첫 번째는 웹 브라우저가 HTML의 소스 코드만 보고도 어느 부분이 제목인지, 어느 부분이 메뉴이고 본문 인지를 쉽게 인식하기 때문이다.
글을 소리로 알려주는 스크린 리더기가 그 예시 인데, 시각 장애인이 사이트의 구조를 소리로 구분할 수 있도록 도와준다.
두 번째는 검색 엔진이 검색을 수행할 때에는 HTML 내에 있는 태그를 분석하는데, 그 태그들의 의미가 분명하다면 검색 엔진 입장에서 구분하기가 쉬워지기 때문이다.
세 번째는 코드 가독성 및 유지 보수가 쉬워지기 때문이다. 만약 모든 태그들이 div로 되어있을 경우 어느 부분이 어느 영역인지 파악하기가 어렵다.
하지만 시맨틱 태그를 이용하면 유지 보수를 할 때나 다른 작업자가 코드를 파악하고 접근하기가 쉬워진다.
시맨틱 태그 사용 이전에는 각 부분을 지칭하는 이름이 통일되지 않아 웹 문서 내용을 검색하는 검색 엔진에게도,
본인 것이 아닌 작업을 진행하는 작업자에게도 구분에 있어 어려움이 있었지만,
시맨틱 태그 사용 이후 부터 이러한 어려움이 없어졌다.

2-2. 시맨틱 태그의 용도

  • header : 헤더 영역을 의미하는 태그로, 주로 사이트 제목이나 로고, 검색 창이 위치한다. 주로 맨 위나 왼쪽에 온다.
  • main : 웹 문서의 메인 컨텐츠가 들어가는 영역이다. 보통 웹 사이트 마다 그 내용은 상이하며 메인 태그는 웹 문서에서 한번만 사용할 수 있다.
  • footer : 푸터 태그는 웹 문서에서 맨 아래쪽에 위치할 컨텐츠를 담는 영역이다. 주로 회사 소개나 사이트 정보, sns안내 등의 메뉴가 담긴다.
    푸터 영역에는 header 태그나 section 태그, article 태그를 모두 사용할 수 있다.
  • nav : nav태그는 navigation link의 집합을 정의할 때 사용하는 태그이다.
    이 태그를 통해 다른 페이지나 현재 페이지에서 다른 부분과 연결되는 링크를 만들 수 있다.
  • section : section태그는 article 태그와 비슷해 보이지만 다르다.
    웹 문서의 콘텐츠 영역을 나타내며 article과 달리 다수의 콘텐츠를 묶는 용도로 사용된다.
  • article : article태그는 section태그가 여러 컨텐츠를 묶는 것과 달리, 신문 기사처럼 단독적인 콘텐츠를 담는다.
    한 웹 문서 안에 여러 개 쓸 수 있으며 이 안에 섹션 태그를 넣을 수도 있다.
  • aside : aside태그는 웹 문서 작성에 있어 필수적인 요소는 아니다.
    본문에 해당하지 않고 사이드 바에 해당하는 내용을 담기 때문이다.
